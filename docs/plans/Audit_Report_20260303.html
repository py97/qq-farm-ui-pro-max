2. 发现的重大缺陷 (Critical Issues)

BUG Backend - farm.js 方法调用错误

位置： `core/src/services/farm.js` (checkFarm)

描述： 错将 `farmScheduler.has(taskKey)` 写成了 `farmScheduler.hasTask(taskKey)`。由于 `hasTask` 不存在，会导致异常被 catch 且任务调度异常。

影响： 高 - 农场自动巡查可能因调度冲突或重复执行而失效。

SEC Backend - admin.js 权限绕过风险

位置： `core/src/controllers/admin.js`

描述： `/api/account-logs` 接口遗漏了 `authRequired` 中间件校验。

影响： 中 - 非法用户可能通过构造请求读取到系统级敏感操作日志。

RISK Backend - scheduler.js 漂移补偿竞争风险

位置： `core/src/services/scheduler.js` (setIntervalTask)

描述： 在 `preventOverlap` 模式下，底层 `expectedNext` 的更新早于异步任务实际执行时刻。若任务执行时间极长，可能导致后续定时触发基准偏移。

3. 前端优化建议 (Frontend Audit)

  • setting.ts: `SettingsState` 接口定义缺失 `stakeoutSteal` 字段，导致代码中出现大量 `as any`。建议补全接口定义以增强类型安全。
  • Dashboard.vue: 显示金币/点券变动时，建议增加颜色对比度。当前 `text-green-500` 在浅色模式下较清晰，但在深色模式下建议调优。
