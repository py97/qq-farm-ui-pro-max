五、优化建议

建议 1：把“账号失效”从临时状态升级为正式业务状态

  • 当前已经在运行态中正确停机并保留 wsError=400，下一步建议在账号表中引入明确的 status=expired_auth 或等效字段。
  • 这样可以避免服务重启后只剩日志、不剩结构化状态，也便于过滤、统计和批量提醒。

建议 2：把 Redis 降级做成显式配置开关

  • 现在系统已经能在 Redis 鉴权失败时自动降级，但更稳妥的做法是在环境变量中增加“明确禁用 Redis”的选项。
  • 这样部署时不需要靠报错再降级，启动日志也更干净。

建议 3：补自动化回归

  • 优先补以下自动化用例：空用户名登录、普通用户路由越权、普通用户保存管理员设置、Redis 启动降级、WS400 后账号停机与提示。
  • 这些都是已经确认过的高风险回归点，适合最先纳入 API 集成测试和前端 E2E。

建议 4：清理前端构建告警

  • 本轮功能修复完成后，前端仍有两类非阻断告警：失效图标名、动态导入与静态导入混用。
  • 它们不会阻塞当前版本使用，但会增加后续维护成本，建议单独做一次前端构建治理。
