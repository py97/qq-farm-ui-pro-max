# 全局功能优化后续审查与影响评估 (v4.2.0)
发布日期：2026-03-05

## 1. 对主线业务的健康度安全检查
在完成今日 T1 至 T11 即“安全与监控增强 / 全局通用方法封装 / 数据雷达”的过程中，我们重点针对以下环节进行了副作用复核：

### 1.1 访客行为检测 (关于产生过多请求的隐患)
*   **实现机制检查**：`detectAndLogVisitorChanges` 方法是在**地块数据被拿回来之后**通过纯前端对象比对执行的（利用 `JSON.stringify` 或对象比对校验状态）。
*   **影响评估 (安全)**：由于它使用的是从 `landsReply` 当中已经取回的现成信息并缓存在本地内存 `visitorCache`，**完全不会向腾讯服务器发送额外查询包**。因此，它的启用不会触发防刷机制。
*   **遗留问题/边界现象**：如果系统 Node 进程重启，`visitorCache` 将被销毁。那么系统拉取第一波农场数据时，将建立一次底模，此时会不触发任何提醒。这是合理的预期行为，无负面影响。

### 1.2 策略图鉴 (内存资源消耗)
*   **计算资源复盘**: 在前端 `Analytics.vue` 获取种子详情时进行的排序、四大特征点提取是在 Vue 的 computed 或 watcher 中实时映射的。
*   **评价**: 虽然农场中总作物超过数百个，但采用现代浏览器的 V8 VDOM 运算仅需数毫秒。而且使用 Vue 3 核心组合式函数进行了防抖限截，不会造成额外的内存泄漏。

## 2. 目前已知的技术债与未来的优化建议
在此次大版本的功能并入中，虽然我们将新版逻辑完全打通，但同时引入了“降维改进”的契机：

**建议 1: 全局复用 `common.js` 中的高阶异步锁 (重点)**
*   **现状**: 虽然我们在 T4 中成功封装了来自 PR 版的强大 `common.js`（包括 `withTimeout`, `createDailyCooldown`, `withRetry` 等）。
*   **动作空间**: 我们目前尚未对系统现有的 `core/src/services/` (如 `farm.js`、`friend.js`) 的所有底层网络包裹方法全部进行强制解耦和映射！
*   **下一步**: 建议在下个阶段，对 `farm.js` 中那些生硬手写的定时 `sleep` 以及可能超时的循环，全面换绑成 `common.js` 定义的高阶函数包装器，从根源上将项目的工程质量提升至企业中间件级别。

**建议 2: 关于后台守护进程的跨平台冲突**
*   **发现问题**: 本次新增的 `systemd`, `pm2` 等开箱即用脚本多为针对 Linux 等级服务器研发设计的。由于 Mac 开发环境没有 `systemd` (Mac 采用 `launchd`)，所以直接在 Mac 执行 `./install-systemd-service.sh` 必然报错。
*   **保障机制已建立**: 脚本已经内部置入了 `OS` 鉴定拦截（如检测不是 Linux 直接 exit 0）。如果部署在甲骨文 CentOS、或者是 Ubuntu，它才会工作。这符合预期。

**建议 3: UI 全量适配移动端的最后巡查**
*   现在新增加的“策略面板推荐”以及“高级统计区”，在宽度特别窄的 iPhone 屏幕下，可能会被迫引发横向滑动。可以在随后通过 `UnoCSS` 加入更细颗粒的手机端 `hidden` 或折叠抽屉来实现更美的降级展示。

## 3. 评估结论
**一切良好！** 本次 T1 - T11 重构并未修改任何能够突破当前稳定 `防偷与化肥保护体系` 的调度底座，属于非破坏性平滑升级。日志体系非常健康，通知准确。架构完全准备好可以供用户正式操作。
